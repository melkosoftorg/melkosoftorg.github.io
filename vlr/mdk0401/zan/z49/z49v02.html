назад

Задание на практическую работу:

Задачи:

  1. Установка ACT 5.6.
  2. Настройка.
  3. Создать коллекцию.
  4. Анализ логов.

Требования:

  1. Установленная клиентская операционная система (одна из перечисленных):
  • Windows 7
  • Windows Vista SP1 или SP2
  • Windows Server 2008 R2
  • Microsoft Windows Server 2003 SP1 или SP2
  • Microsoft Windows XP c Service Pack 2 или SP3
  • Microsoft Windows 2000 SP4 и Update Rollup 4.
  1. База данных SQL:
  • Microsoft SQL Server 2008
  • SQL Server 2005
  • SQL Server 2008 Express
  • SQL Server 2005 Express Edition
  1. .Net Framework 3.5.
  2. Windows Installer 4.5.
  1. Процессор с тактовой частотой 2.8 ГГц и 2 Гб оперативной памяти

Задача№1.

  1. Для начала необходимо установить SQL Server и при необходимости .Net Framework 3.5.
  2. Запускаем установочный файл ApplicationCompatibilityToolkitSetup.exe.
  3. Нашагеприветствия "Welcome to the Microsoft Application Compatibility Toolkit 5.6 Setup Wizard" жмем "Next".
  4. Принимаемлицензионноесоглашение, поставивгалочкунапротив "I accept the terms in the License Agreement". Затемжмем "Next".
  5. Параметры на шаге "Destination Folder" оставляем без изменений. Жмем "Next".
  6. Нашаге "Ready to install Microsoft Application Compatibility Toolkit 6.5" жмем "Install".
  7. По окончании успешной установки жмем "Finish".

Задача№2.

  1. Запускаем Application Compatibility Manager (ACM), дляэтогожмемПуск -> Всепрограммы -> Microsoft Application Compatibility Toolkit -> Application Compatibility Manager.
  2. Нашаге "Welcome to the ACT Configuration Wizard" жмем "Next".
  3. На следующем шаге выбираем "Enterprise Configuration" и жмем "Next".
  4. Далее в строке "SQL сервер" указываем имя нашего SQL сервера и жмем "Connect". В строке "Database" указываем имя создаваемой базы, например "firstdb" и жмем "Create". После успешного создания базы жмем "Next".
  5. На шаге "Configure Your Log File Location" указываем местоположение папки, в которой будут храниться логии клиентов, нажатием кнопки "Browse". К примеру, это будет папка C:\LogFiles. Параметр "Share as" конфигурируется автоматически, при необходимости можем изменить его. Жмем "Next".
  6. Далее необходимо указать имя учетной записи, под которой будет осуществляться запуск службы ACT Log Processing Service. Оставляем без изменения, т.е. используется пользователь "Local System". Жмем "Next".
  7. На завершающем шаге при желании снимаем галочку "Send ACT usage data to Microsoft" и жмем "Finish".

Задача №3.

  1. Вглавномокне "Application Compatibility Manager Overview" жмем "Collect – Create your data collection package".
  2. В открывшемся окне делаем следующие настройки:
  • Name: All Collect;
  • Evaluate compatibility when: Deploying a new Operation System;
  • Advanced: оставляем без изменений
  • Starting: As soon as possible after installation;
  • Duration: 3 days;
  • Upload: ever 2 hours;
  • Output Location: общая папка на сервере;
  • Label Collection Data: MS Word 2007;
    3. Поокончании настройки коллекции жмем File -> Save and Create Data Collection Package. Указываем имя "All_Collectors.msi".
    4. Устанавливаем пакет любым из возможных способов. Для осуществления тихой установки воспользуемся ключом /qn.

Задача №4.

  1. После установки пакета на клиенте происходит автоматическое сканирование. Логи сохраняются в папку, сконфигурированную во время создания пакета.
  1. На сервере ACT в Application Compatibility Manager в нижней части правой колонки выбираем "Analyze".
  2. Переходим в раздел "Windows 7 Reports".
  3. Выбираем "Applications".
  4. Выбираем необходимое приложение. Жмем правой кнопкой мыши и выбираем "Set priority".
  5. Выбираем "Priority 1 – Business Critical" ижмем "Ok".
  6. При возникновении каких-либо проблем дважды жмем на приложение.
  7. В открывшемся окне выбираем вкладку "Issues" и знакомимся с причинами несовместимости.
  8. По каждой причине можно получить дополнительную информацию, просто выбрав ее.
