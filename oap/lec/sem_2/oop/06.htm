Абстрактные классы

Абстрактным классом называется класс, в котором есть хотя бы одна чистая (пустая) виртуальная функция.

Чистая виртуальная функция - компонентная функция, которая имеет определение: 

	virtual тип имя_функ(список_формальных_парам) = 0;

где      = 0 - "чистый спецификатор".

Пример:     virtual void fpure(void) = 0;

Чистая виртуальная функция

  • ничего не делает и недоступна для вызовов;
  • предназначена служить основой для подменяющих ее функций в производных классах, невозможно создать самостоятельные объекты абстрактного класса,
  • производные классы должны обеспечить определение функции для каждой чистой виртуальной функции базового класса.

Абстрактный класс может использоваться только в качестве базового для производных классов, чтобы построить иерархию классов и определить общие свойства для производных классов.
При создании объектов такого производного класса в качестве подобъектов создаются объекты абстрактного базового класса.
Имеется абстрактный класс B         Производные от класса B классы: 
class В
{	protected:
		virtual void func(char) = 0;
		void sos(int);
};
    
class D : public В
{	...
	void func(char);
};
class E : public В
{
	void sos(int);
};
В классе D чистая виртуальная функция func() заменена конкретной виртуальной функцией того же типа. Функция B :: sos() наследуется классом D и доступна в нем и в его методах.
Класс D не абстрактный.

В классе E переопределена функция B :: sos(), а виртуальная функция B :: func() унаследована. Тем самым класс B становится абстрактным и может использоваться только как базовый.

Как всякий класс, абстрактный класс может иметь явно определенный конструктор. Из конструктора возможен вызов методов класса но любые прямые или опосредованные обращения к чистым виртуальным функциям приведут к ошибкам во время выполнения программы.

Правила:

  • абстрактный класс не может использоваться в качестве типа аргумента функции или типа возвращаемого значения;
  • абстрактный класс нельзя использовать в явном преобразовании;
  • нельзя определить представитель абстрактного класса (локальную/глобальную переменную или элемент данных);
  • можно определять указатель или ссылку на абстрактный класс;
  • если класс, производный от абстрактного, не определяет все чистые виртуальные функции абстрактного класса, он также является абстрактным.

Пример использования абстрактных классов - предоставление интерфейса с полным отсутствием деталей реализации.
Например, операционная система может скрыть детали реализации драйверов устройств за интерфейсом абстрактного класса.
