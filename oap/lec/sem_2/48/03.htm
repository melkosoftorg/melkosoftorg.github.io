Глобальные идентификаторы Borland C++

Рассмотрим теперь глобальные идентификаторы Borland C++ (макроимена):

  • __BCPLUSPLUS__ - определен при компиляции программ на языке Borland C++. Задает версию системы, например, для системы программирования Borland C++ 3.0 имеет значение 0 х 0300, для системы программирования Borland C++ 3.1 имеет значение 0 х 0310.

  • __CDECL__ - определен, если используются соглашения, принятые в языке Си.

    Система программирования Borland C++ поддерживает шесть различных моделей памяти:

    • tiny (минимальная),
    • small (малая),
    • medium (средняя),
    • compact (компактная),
    • large (большая),
    • huge (максимальная).

  • __TINY__, __SMALL__, __COMPACT__, __MEDIUM__, __LARGE__, __HUGE__ - позволяют установить модель памяти, заданную пользователем в Borland C++. При конкретной компиляции определено только одно из этих макроимен.

  • __cplusplus - определен при использовании компилятора языка C++ (принимает значение 1); 
    	// Если программа не будет откомпилирована средствами языка C++,
	// вывод значений осуществится с помощью функции printf().
	#ifndef __cplusplus
		#include<stdio.h>
	#else
		#include<iostream.h>
	#endif
	void main(void)
	{	#ifdef __cplusplus
			cout << "1 - Hello " << endl;
		#else
			printf("2 - Hello \n");
		#endif
	}

  • __DLL__ - определен, если используются динамически связанные библиотеки;

  • __MSDOS__ - дает целочисленную константу для всех случаев компиляции;

  • __OVERLAY__ - определен, если включена поддержка оверлейных структур;

  • __PASCAL__ - определен, если используются соглашения, принятые в языке Pascal;

  • __TCPLUSPLUS__ - принимает те же значения, что и _BCPLUSPLUS;

  • __TEMPLATES__ - определен, если поддерживается оператор template;

  • __TURBOC__ - дает номер текущей версии Borland C++ в виде шестнадцатеричной константы (для системы программирования Borland C++ 3.1 эта константа будет равна 0х0410);

  • __WINDOWS__ - определен, если генерируется код, специфичный для системы Windows.
