Оператор goto

goto метка

Оператор goto – оператор безусловного перехода, осуществляет переход по метке.

Как только в программе встречается оператор goto, сразу же выполняется переход (прыжок) к месту программы, обозначенному в этом операторе. Переход называется безусловным, потому что не зависит ни от каких условий и всегда выполняется при достижении оператора goto.

Именем метки может выступать любой идентификатор, не использующийся для объявления переменных, констант, типов или функций. Метка указывается на новой строке с завершающим двоеточием. После оператора goto указывается имя метки с завершающей точкой с запятой.
Все метки должны быть уникальны и не могут повторяться. 

	#include<conio.h>
	#include<stdio.h>
	void main()
	{	 	int n; 
		start:	clrscr();
			puts("Введи число от 0 до 10"); scanf("%d", &n);
			if (n < 0 || n > 10)
				goto start;
			if (n == 0)
				goto num0;
			else if (n == 1)
				goto num1;
			else
				goto num2;
		num0:	puts("Число 0\n"); goto end;
		num1:	puts("Число 1\n"); goto end;
		num2:	puts("Число > 1\n");

		end:		getch();
	}

Пример: Вычислить сумму первых N членов ряда 1/N. 

	#include<stdio.h>
	void main()
	{
 		  int N;
		  float S = 0.0;
		  printf(“Введите N:”); scanf(“%d”, &N);
	LABEL:
		  S += 1.0/N--;
		  if(N > 0) goto LABEL;
		  printf(“S = %.2f\n”, S);
	}

Наиболее разумное решение при программировании - обойтись без использования оператора goto.

Иногда в литературе структурное программирование называют программированием без goto. Действительно, при таком подходе нет места безусловному переходу. Неоправданное использование в программах оператора goto лишает ее структурности, а значит, всех связанных с этим положительных свойств: прозрачности и надежности алгоритма.

Замечание: хотя оператор goto присутствует во многих современных языках программирования высокого уровня, но он считается устаревшим оператором и его использование не рекомендуется.
