Автоматические переменные

По умолчанию переменные, описанные внутри функции, являются автоматическими, что можно подчеркнуть явно с помощью ключевого слова auto: 
		main()
		{
			auto int pl;
			. . .
		}

Автоматические переменные имеют локальную область действия, начинают существовать при вызове функции, содержащей ее.
Когда функция завершает свою работу и возвращает управление туда, откуда ее вызвали, автоматическая переменная исчезает, ячейка памяти может снова использоваться для чего-нибудь другого.

Область действия автоматической переменной - область, ограниченная блоком { }, в котором переменная описана.

Глобальные данные НЕ могут быть автоматическими, потому что нет блока, которому они принадлежат.

Вследствие того, что автоматические переменные имеют смысл только внутри собственной функции, можно использовать одно и то же имя переменной в разных функциях. 

void main()
{
	int age;
	printf("Введите Ваш возраст: ");
	scanf("%d", &age);
	spouse();
	printf("Ваш возраст составляет %d лет", age);
}
void spouse()
{
	int age;
	printf("Введите возраст Вашего супруга(и): ");
	scanf("%d", &age);
	printf("Возраст Вашего супруга(и) составляет %d лет\n", age);	
}

Переменная с именем age определена и в функции main(), и в функции spouse(). Каждая переменная является автоматической (или локальной) для той функции, где ее определили, так что на самом деле в программе присутствуют две переменные с одинаковым именем. Каждая из них имеет собственное значение, которое хранится в различных элементах памяти. Компилятор определяет, значение какой из переменных age должно быть отображено на экране, на основании того, в какой из функций помещена инструкция printf().
