Какой класс памяти применять?

Ответ на вопрос почти всегда один - "автоматический".

Одно из золотых правил защитного программирования: "Необходимо знать только то, что нужно".

Организуйте работу каждой функции автономно, насколько это возможно, и используйте глобальные переменные только тогда, когда это действительно необходимо.

Резюме: Классы памяти.

  1. Ключевые слова: auto, extern, static, register.

  2. Общие замечания:

    Класс памяти

    • определяет область действия переменной и продолжительность ее существования в памяти,
    • устанавливается при описании переменной с соответствующим ключевым словом.

    Переменные, определенные вне функции, являются внешними и имеют глобальную область действия.

    Переменные, определенные внутри функции, являются автоматическими и локальными, если только не используются другие ключевые слова.

    Внешние переменные, определенные раньше функции, доступны ей, даже если не описаны внутри нее.

  3. Свойства:

    Класс памяти
    Ключевое слово
    Время жизни
    Область действия
    Описываются
    Автоматический auto временно локальная внутри функции
    Регистровый register временно локальная внутри функции
    Статический static постоянно локальная внутри функции
    Внешний extern постоянно глобальная (все файлы) вне функции
    Внешний статический static постоянно глобальная (1 файл) вне функции
