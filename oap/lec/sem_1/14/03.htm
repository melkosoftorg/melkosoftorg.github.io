Возможность доступа к файлу

access - определяет возможность доступа к файлу, проверяет файл с именем filename для определения, существует ли он, может ли он быть прочитан, записан или выполнен.
Синтаксис int access(char * filename, int amode);
Прототип в io.h

Список значений параметра amode
06- проверка разрешения на чтение и запись,
04- проверка разрешения на чтение,
02- проверка разрешения на запись,
01- проверка на возможность выполнения,
00- проверка на существование файла.

Под управлением операционной системы MS-DOS все существующие файлы имеют доступ на чтение (amode = 04), поэтому 00 и 04 дают один и тот же результат.
Значения параметра amode 06 и 02 эквивалентны, поскольку, под управлением MS-DOS, доступ на запись подразумевает доступ на чтение.

Если параметр filename имеет ссылку на каталог, функция access просто проверяет, существует данный каталог или нет.
Если требуемый способ доступа разрешен, возвращается значение 0, иначе возвращается значение -1, и переменная errno получает одно из значений:

  • ENOENT траектория доступа (PATH) или имя файла не найдено,
  • EACCES отказ доступа. 
	#include<stdio.h>
	#include<io.h>
	#include<conio.h>
	main()
	{
		char *ch="c:\\progr\\calc.cpp";		//Маршрут поиска файла
		clrscr();
		printf("Файл %s - %s\n", ch, (access(ch, 0) == 0) ? "Да" : "Нет");
		getch();
	}

	Результаты могут быть, если файл существует: 
			Файл c:\progr\calc.cpp - Да
	или, при отсутствии файла:
			Файл c:\progr\calc.cpp - Нет
