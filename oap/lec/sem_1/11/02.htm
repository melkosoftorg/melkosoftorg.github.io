Зачем мы описываем и определяем функцию

В C можно и описывать и определять функцию.
  • Когда вы описываете функцию, то даете всем остальным программам (включая главный модуль main) информацию о том, каким образом должно осуществляться обращение к этой функции.

    Строки программы с описанием функций вводятся для того, чтобы описать как тип функций, так и порядок следования их параметров с целью избежания ошибок.
    Описание функции иначе называют прототипом функции. Ниже приведен прототип функции cube(), который дает следующую информацию:

    • имя функции - cube,
    • в нее необходимо передать один параметр - значение x типа long,
    • результат будет возвращен типа long. 
    		long cube(long x);

    Можно использовать два различных стиля описания функций как классический стиль, который нашел широкое применение в большинстве программ на C, так и современный.

  • Когда вы определяете функцию, вы присваиваете ей имя, по которому к ней будет осуществляться обращение, и указываете, какие конкретно действия она будет выполнять.

    Определение функции - это полный текст функции, включающий весь ее выполняемый код. Первая строка - заголовок функции, полностью совпадающий с прототипом функции.
    Тело функции заключено в фигурные скобки и содержит программный код в данном примере вычисления куба заданного числа. 

    		long cube(long x)
		{
			long v_cub;
			v_cub = x * x * x;
			return(v_cub);
		}
