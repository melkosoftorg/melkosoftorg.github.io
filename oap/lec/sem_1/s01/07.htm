Этап 7.   Решение задачи на ЭВМ,
обработка и оформление результатов счета


На этом этапе ЭВМ выполняет все предусмотренные программой вычисления и выдает результаты.

При проектировании программы надо предусматривать вывод результатов с пояснениями, каким переменным соответствуют выводимые значения:

  • таблицам результатов должны предшествовать заголовки,
  • одни группы данных отделять от других пропусками строк,
  • при табулировании функций выводить и значения аргументов, перед выводом результатов,
  • выводить данные, характеризующие условия эксперимента.

При исполнении разработанной программы мы можем получить в качестве результата или значения искомых величин, если треугольник с такими размерами сторон существует 

         float a = 5.98
         float b = 7.23
         float c = 4.77

         P = 17.98
         s = 14.18
         ha = 4.74     hb = 3.92       hc = 5.94

или сообщение о невозможности существования треугольника с заданными размерами сторон. 

         float a = 2.4
         float b = 4.8
         float c = 7.96

         Недопустимые параметры треугольника

Многократное использование разработанной программы для решения различных задач заданного класса требует проведения следующих дополнительных работ:

  • исправление обнаруженных ошибок;
  • модификация программы для удовлетворения изменяющихся эксплуатационных требований;
  • доработка программы для решения конкретных задач;
  • проведениe дополнительных тестовых просчетов;
  • внесение исправлений в рабочую документацию;
  • усовершенствование программы и так далее.

Применительно ко многим программам работы по сопровождению поглощают более половины затрат, приходящихся на весь период времени существования программы (начиная от выработки первоначальной концепции и кончая моральным ее устареванием) в стоимостном выражении.
Программа, предназначеная для длительной эксплуатации, должна иметь соответствующую документацию и инструкцию по её использованию.

Сопровождение программы включает:

  • доработка программы для решения конкретных задач;
  • составление документации к pешенной задаче, к математической модели, к алгоpитму, к пpогpамме, к набору тестов, к использованию.
