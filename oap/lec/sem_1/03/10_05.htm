Команда Linker (Компоновщик)

Команда Options\Linker (Параметры\Компоновщик) позволяет задать несколько параметров, которые оказывают влияние на процесс компоновки. Команда Linker (Компоновщик) открывает блок диалога, содержащий несколько триггерных кнопок и селективных кнопок.

Блок диалога Settings... ()

Map File (Файл карты отображения)

(∙) Off (Не создается)
( ) Segments (Сегменты)
( ) Publics (Глобальные)
( ) Detailed (Подробный)

Селективные кнопки Map File (Файл карты отображения) следует использовать для выбора типа файла карты отображения, который должен создаваться.
Если параметр отличен от Off (Не создается), то файл карты отображения помещается в директорию результирующих файлов, которая задана в блоке диалога Options\Directories\Output Directiry.
По умолчанию данный параметр имеет значение Off (Не создается).
Можно задать тип генерируемого компоновщиком файла карты, указав "Segments" (для сегментов памяти), "Publics" (для глобальных переменных) или "Detailed" (подробный).

Options (Параметры)

[   ] Initialize segments (Инициализировать сегменты)
[X] Default libraries (Стандартные библиотеки)
[   ] Pack code segments
[   ] Warn duplicate symbols (Дублирующиеся идентификаторы)
[X] "No stack" warning (Предупреждение "Стек отсутствует")
[X] Case-sensitive link (Чувствительность к регистру букв при компоновке)
[   ] Case-sensitive exports
[   ] Compress debug info
  • Маркирование Initialize segments (Инициализировать сегменты) указывает компоновщику инициализировать неинициализированные сегменты. (Обычно необходимость в этом не возникает, задание параметра приводит к увеличению размера выполняемого файла.)
  • Если осуществляли компоновку с модулями, которые не были созданы с помощью компилятора Borland C++, то использованный компилятор мог занести в объектный файл перечень стандартных библиотек.
    Если Default libraries (Стандартные библиотеки) маркирована, компоновщик пытается обнаружить все неизвестные ему подпрограммы в этих библиотеках. А также в стандартных библиотеках, которые входят в состав системы Borland C++.
    Если кнопка не маркирована, компоновщик производит поиск только в стандартных библиотеках, которые входят в состав системы Borland C++, и игнорирует любые ссылки на стандартные библиотеки, которые он обнаруживает в объектных файлах.
  • Маркирование Pack code segments ()
  • Параметр Warn duplicate symbols (Предупреждение о дублирующихся идентификаторах) определяет, будет ли компоновщик предупреждать вас о наличии ранее встретившихся ему идентификаторов в файлах библиотек (с расширением имени .LIB). По умолчанию эта кнопка не маркирована.
  • Параметр "No stack" warning (Предупреждение "Стек отсутствует") определяет, будет ли компоновщик выдавать предупреждающее сообщение "No stack" ("Стек отсутствует").

    Важное замечание: Если выдача данного сообщения не заблокирована, то для программы, генерируемой с заданием минимальной (tiny) модели памяти появление этого предупреждения является нормальным фактом.

  • Параметр Case-sensitive link (Чувствительность к регистру букв при компоновке) определяет, будет ли компоновщик различать регистр букв в идентификаторах. Как правило, данный параметр должен быть маркирован, поскольку C является языком, чувствительным к регистру букв.

Блок диалога Libraries... (Библиотеки)

Libraries (Библиотеки)

[X] Graphics library (Графическая библиотека)
[   ] Turbo Vision (Turbo Vision библиотека)
  • Параметр Graphics library (Графическая библиотека) управляет автоматическим поиском графической библиотеки BGI (Borland Graphics Interface (Графический интерфейс корпорации Borland)). Если эта кнопка маркирована, то можно создавать и выполнять состоящие из единственного файла графические программы, не прибегая к использованию файла проекта. Если эта кнопка не маркирована, то несколько ускоряется процесс компоновки, поскольку компоновщик не производит поиск файла графической библиотеки BGI.

    Важное замечание: Можно демаркировать эту кнопку и по-прежнему создавать программы, в которых используются графические возможности интерфейса BGI, при условии, что добавляете имя графической библиотеки BGI (с именем GRAPHICS.LIB) к списку файлов проекта.

  • Параметр Turbo Vision (Turbo Vision библиотека) подключает заголовочные файлы, охватывающие соответствующие группы стандартных классов Turbo Vision, что дает возможность воспользоваться полной объектно-ориентированной библиотекой классов, которая содержит компоненты пользовательского интерфейса: многочисленные, сворачивающиеся, перекрывающиеся окна; вертикальные меню; поддержку "мыши"; панели диалога; встроенную установку цвета; кнопки, полосы прокрутки, панели ввода, кнопки управления с независимой и зависимой фиксацией; стандартную обработку нажатий клавиш и работы манипулятором "мышь".
