Compile - меню компиляции
Вызов: Alt + C

Меню Compile (Компиляция) позволяет осуществлять:
  • компиляцию программы в объектный файл,
  • избирательную рекомпиляцию в выполняемый файл,
  • компоновку exe - файла,
  • полную рекомпиляцию всех файлов,
  • получение информации о текущем файле,
  • удаление сообщений.

Для того, чтобы использовать команды Compile (Компилировать), Make (Избирательная рекомпиляция), Link (Компоновка) и Build all (Полная рекомпиляция), необходимо, чтобы в активном окне редактирования был бы открыт какой-либо файл или чтобы был задан какой-либо проект (для команд Make (Избирательная рекомпиляция), Build (Полная рекомпиляция) и Link (Компоновка)).
Например, если вы открываете окно Message (Сообщение) или Watch (Просмотр выражений), указанные команды меню будут "запрещены".

 

Команда Compile (Компилировать в объектный файл)
Вызов: Alt + F9

Команда Compile\Compile (Компиляция\Компилировать в объектный файл) осуществляет компиляцию активного файла редактора (файла с расширением *.C или *.CPP в файл с расширением *.OBJ).
Это меню всегда отображает имя того файла, который должен быть создан в результате.

Когда система Borland C++ выполняет компиляцию, на экран выдается блок статуса, в котором отображается прохождение процесса компиляции и результаты.
Когда будет завершена компиляция/компоновка, для удаления с экрана данного блока следует нажать произвольную клавишу.
Если возникают какие-либо сообщения об ошибках или предупреждающие сообщения, активным становится окно Message (Сообщение), в котором отображается и выделяется световым маркером первое сообщение об ошибке.

 

Команда Make (Избирательная рекомпиляция в выполняемый файл)
Вызов: F9

Команда Compile\Make (Компиляция\Избирательная рекомпиляция в выполняемый файл) вызывает Менеджер проектов для создания EXE-файла.
Данная команда меню всегда отображает имя того EXE-файла, который должен быть создан в результате. Например, EXE file: C:\EXAMPLE.EXE

Имя EXE-файла выбирается на основании одного из двух имен в следующем порядке:

  • имени файла проекта (с расширением имени *.PRJ), который задается с помощью команды Project\Open Project (Проект\Открыть проект);
  • имени файла в активном окне редактирования (если проект не был задан, то будет использоваться стандартный проект, заданный файлом TCDEF.DPR).

Команда Compile\Make (Компиляция\Избирательная рекомпиляция в выполняемый файл) осуществляет рекомпиляцию только тех файлов, исходный код которых обновлялся после создания объектного кода.

 

Команда Link (Компоновать EXE файл)

Команда Compile\Link (Компиляция\Компоновать EXE файл) использует текущий *.OBJ и *.LIB файлы (задаваемые либо по умолчанию, либо в текущем файле проекта) и компонует их, не производя избирательной компиляции; в результате получается новый EXE-файл.

 

Команда Build All (Полная рекомпиляция всех файлов)

Команда Compile\Build All (Компиляция\Полная рекомпиляция всех файлов) осуществляет полную рекомпиляцию всех файлов, составляющих ваш проект вне зависимости от того, как их дата исходного кода соотносится с датой объектного кода.
Данная команда аналогична команде Compile\Make (Компиляция\Избирательная рекомпиляция в выполняемый файл) с тем исключением, что рекомпиляция производится безотносительно к датам исходных файлов.

Команда Compile\Build All (Компиляция\Полная рекомпиляция всех файлов) сначала устанавливает дату и время всех объектных файлов (файлов с расширением имени *.OBJ) для текущего проекта в ноль, а затем осуществляет избирательную рекомпиляцию.
Таким образом, если вы прервете выполнение команды Build All (Полная рекомпиляция всех файлов) нажатием комбинации клавиш Ctrl + Break или столкнетесь с ошибками, которые прервут процесс полной рекомпиляции, вы можете возобновить его с той самой точки, где он был прерван, если выберете команду Compile\Make (Компиляция\Избирательная рекомпиляция в выполняемый файл).

 

Команда Information... (Получить информацию)

Команда Compile\Information... (Компиляция\Получить информацию...) отображает блок, в котором содержится информация относительно текущего файла:
  • Имя стандартной директории.
  • Имя файла в активном окне.
  • Объем расширенной памяти, зарезервированной для использования системой Borland C++.
  • Объем отображаемой памяти, зарезервированной для использования системой Borland C++.
  • Число скомпилированных строк.
  • Число выданных системой предупреждающих сообщений.
  • Число сгенерированных ошибок.
  • Объем времени, затраченного на последнее выполнение программы.
  • Статус отладки.
  • Код завершения программы в операционной системе, который был получен при последнем завершении выполнения программы.
  • Объем доступной памяти операционной системы.
  • Объем времени, затраченного на выполнение последнего шага отладки.

Представленная в этом блоке информация предназначена только для отображения; вы не сможете модифицировать какие-либо значения, представленные в нем. После просмотра информации, необходимо нажать клавишу Enter, чтобы покинуть блок диалога.

 

Команда Remove Messages (Удалить сообщения)

Команда Compile\Remove messages (Компиляция\Удалить сообщения) удаляет все сообщения из окна Message (Сообщение).
