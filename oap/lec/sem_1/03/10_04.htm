Options - меню параметров
Вызов: Alt + O

 

Команда Make... (Избирательная рекомпиляция)

Команда Options\Make... (Параметры\Избирательная рекомпиляция) отображает блок диалога, который позволяет задать условия, в которых будет осуществляться управление проектом.

Break Make On (Прервать избирательную рекомпиляцию на)

( ) Warnings (Предупреждающих сообщениях)
(∙) Errors (Ошибках)
( ) Fatal errors (Неисправимых ошибках)
( ) All sources processed ()

Селективные кнопки Break Make On (Прервать избирательную рекомпиляцию на) используются для задания того условия, которое будет приводить к прекращению процесса создания проекта. По умолчанию работа прекращается после компиляции файла, в котором содержатся ошибки.

After Compiling ()

( ) Stop
(∙) Run linker
( ) Run librarian

Generate Import Library ()

( ) No
(∙) Use DLL file exports
( ) Use DEF file exports

[X] Check auto-dependencies (Проверка последовательностей зависимости)

Если маркирована Check auto-dependencies (Проверка последовательностей зависимости), менеджер проектов автоматически проверяет зависимости для каждого объектного файла на диске, которому соответствует исходный файл с расширением имени .CPP в списке проекта.
Менеджер проекта открывает объектный файл и ищет информацию о файлах, включенных в исходный код. Эта информация всегда заносится в объектный файл при компиляции исходного модуля как интегрированной версией компилятора Borland C++, так и компилятором командной строки.
Затем проводится сверка времени и даты создания каждого файла, который был использован для построения объектного файла, с информацией о дате и времени, которая хранится в объектном файле. Если даты различны, осуществляется рекомпиляция исходного файла с расширением имени .CPP.
Этот процесс носит название автоматической проверки зависимостей. Если триггерная кнопка не маркирована, никакой проверки файлов не делается.
