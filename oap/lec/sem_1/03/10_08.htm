Options - меню параметров
Вызов: Alt + O

 

Команда Directories... (Директории)

Команда Options\Directories.. (Параметры\Директории) позволяет указать системе Borland C++, где она должна осуществлять поиск файлов, необходимых для компиляции и компоновки, и куда должны направляться двоичные файлы и файлы карты отображения.

При выборе данной команды открывается блок диалога, в котором содержится четыре блока ввода:

  • Блок ввода Include Directories (Директории для включаемых файлов) задает ту директорию, в которой содержатся используемые включаемые файлы.
    Стандартными включаемыми файлами являются те файлы, имена которых предстают заключенными в угловые скобки <> в операторе #include.
    Например,    #include <myfile.h>
    Кроме того, в этих директориях осуществляется поиск заключенных в символы кавычек имен включаемых файлов, которые не были обнаружены в текущей директории.
    Здесь можно задавать имена нескольких директорий, при этом они должны быть разделены символами точки с запятой.

  • Блок ввода Library Directories (Директории для файлов библиотек) задает те директории, в которых содержатся стартовые объектные файлы (startup object files с именами C0?.OBJ), файлы библиотек периода выполнения (с расширениями LIB) и любые другие библиотеки, которые могут использоваться в проекте.

  • Блок ввода Output Directory (Директории для результирующих файлов) задает ту директорию, в которую помещаются файлы с расширениями имен OBJ, EXE и MAP.
    При избирательной компиляции или выполнении программы эта директория используется системой Borland C++ для поиска файлов.
    Кроме того, она используется для проверки даты и времени создания файлов с расширениями OBJ и EXE.
    Если эта строка пуста, то файлы заносятся в текущую директорию.

  • Блок ввода Source Directories (Директория исходных программ)

При вводе имен директорий в эти блоки ввода следует пользоваться следующими правилами:

  • несколько маршрутных имен директорий (если допускается ввод более одного имени) следует разделять символом точки с запятой (;).Введенная строка не должна превышать 127 символов (с учетом символов пробела).
  • пробельные символы перед символом точки с запятой и после нее являются допустимыми, но не обязательными.
  • допускается использование относительных и абсолютных маршрутных имен, в том числе маршрутные имена, которые задаются относительно текущей директории на неактивных дисководах.
    Например, C:\BC\LIB; C:\C\MYLIBS; A:\BC\MATHLIBS; A:..\VIDLIBS
