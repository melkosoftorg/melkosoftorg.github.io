Options - меню параметров
Вызов: Alt + O

 

Команда Environment (Среда работы)

Команда Options\Environment (Параметры\Среда работы) позволяет задать значения параметров, которые будут действительны при работе в любой точке среды функционирования.
Выбор этой команды приводит к открытию меню, которое позволяет выбрать параметры в блоках диалога Preferences (Приоритеты), Editor (Редактор), Mouse (Мышь), Desktop (Макет экрана), Startup (), Colors(Цвет).

Блок диалога Preferences (Приоритеты)

Screen Size (Размер экрана)

(∙) 25 lines (25 строк)
( ) 43/50 lines (43/50 строк)

Группа селективных кнопок Screen Size (Размер экрана) позволяет указать, в каком режиме интегрированная среда будет осуществлять отображение данных на экране - 25 или 43 / 50 строк. В зависимости от типа видеоадаптера на используемом персональном компьютере будут доступны одна или две из этих кнопок. Когда данный параметр установлен в 25 строк (это значение используется по умолчанию), система Borland C++ использует 25 строк и 80 столбцов. Это единственный размер экранного кадра, который доступен на системах с монохромным дисплеем или при работе с цветным графическим адаптером (CGA). Если на персональном компьютере установлена плата EGA или VGA, этому параметру можно задать значение 43/50. Если адаптер EGA, то интегрированная среда отображает 43 строки и 80 столбцов. Если адаптер VGA, то система отображает 50 строк и 80 столбцов.

Source Tracking (Трассировка исходного кода)
(∙) New window (Новое окно)
( ) Current window (Текущее окно)

Если осуществляете пошаговую трассировку исходного кода или хотите просмотреть исходный код, находясь в окне Message (Сообщение), то при попытке просмотра файла, который еще не был загружен, интегрированная среда открывает новое окно. Выбор значения параметра Current window (Текущее окно) приводит к тому, что в подобных ситуациях интегрированная среда будет заменять содержимое окна редактирования, которое находится на самой вершине стека, помещая туда новый файл вместо того, чтобы открывать новое окно редактирования.

Auto Save (Автоматическое сохранение)
[   ] Editor files (Файлы редактора)
[X] Environment (Параметры среды)
[X] Desktop (Макет экрана)
[X] Project (Проект)

Если в блоке триггерных кнопок Auto Save (Автоматическое сохранение) маркирована кнопка Editor files (Файлы редактора), и если с момента последнего сохранения файла на диске он был модифицирован, система Borland C++ будет при выборе команды Run\Run (Выполнить\Выполнить) (или любой команды отладки/выполнения, а также команды File\OS Shell (Файл\Оболочка DOS)) автоматически сохранять исходный файл в окне редактирования. Если маркирована кнопка Environment (Параметры среды),то при выходе из системы Borland C++ будут автоматически сохранены все значения параметров, которые были заданы в этом блоке диалога. Если маркирована кнопка Desktop(Макет экрана), то система Borland C++ определяет, должен ли используемый макет экрана сохраняться при выходе из системы и восстанавливаться при возврате в систему Borland C++. Если маркирована кнопка Project (Проект), система Borland C++ сохраняет всю информацию по проекту, автоматически определяемые зависимости и параметры обработки модулей при выходе из системы и восстанавливает их при возврате в систему Borland C++.

[   ] Save old messages (Сохранять старые сообщения)

Если маркирована кнопка Save old messages (Сохранять старые сообщения), система Borland C++ сохраняет те сообщения об ошибках, которые в настоящий момент находятся в окне Message (Сообщения), добавляя в это окно те сообщения, которые появляются в результате последующих компиляций. Когда происходит компиляция файла, все сообщения по этому файлу удаляются из окна Message (Сообщения), и в его конец добавляются новые сообщения. Если демаркируете эту кнопку, система Borland C++ автоматически стирает сообщения перед компиляцией, избирательной компиляцией или переходом к другой программе.

Блок диалога Editor (Редактор)

Editor Options (Параметры редактора)
[X] Create backup files (Создавать дубликатные файлы)
[X] Insert mode (Режим вставки)
[X] Autoindent mode (Режим автоматических структурных отступов)
[X] Use tab character (Использовать символы табуляции)
[X] Optimal fill (Оптимум заполнения)
[X] Backspace unindents (Удаление структурного отступа при нажатии)
[X] Cursor through tabs (Перемещать курсор по меткам табуляции)
Блок диалога Editor Options (Параметры редактора) содержит несколько триггерных кнопок, которые управляют режимом обработки текста в окнах редактирования.
  • Маркирование Create backup files (Создавать дубликатные файлы - используется по умолчанию) побуждает систему Borland C++ автоматически создавать дубликат исходного файла в окне редактирования при выборе команды File\Save (Файл\Сохранить) и задавать дубликатному файлу расширение имени .BAK.
  • Если кнопка Insert mode (Режим вставки) не маркирована, любой текст, который вводите в окна редактирования, будет замещать существующий текст. Если эта кнопка маркирована, вводимый текст вставляется. Нажатие клавиши Ins при работе в окне редактирования осуществляет триггерное переключение режима вставки.
  • Если маркирована кнопка Autoindent mode (Режим автоматических структурных отступов), нажатие клавиши Enter в окне редактирования приводит к позиционированию курсора под первым не пробельным символом в предшествующей непустой строке. Это средство может оказать неоценимую помощь в придании коду более "читабельного" вида.
  • Если маркирована кнопка Use tab character (Использовать символы табуляции), система Borland C++ при нажатии клавиши Tab (Табуляция) действительно вставляет символ табуляции (код 9 в таблице ASCII). Если кнопка не маркирована, система Borland C++ заменяет символы табуляции на пробелы, число которых определяется значением параметра Tab Size (Размер табуляционной метки).
  • Если маркирована кнопка Optimalfill (Оптимум заполнения), система Borland C++ начинает каждую строку, сформированную на основе принципа автоматической структуризации, внося в нее минимально возможное число символов. При этом по мере необходимости используются символы табуляции и пробела, создаются строки, которые содержат меньшее число символов по сравнению с тем, которое вносилось бы в строку при немаркированном состоянии кнопки Optimal fill (Оптимум заполнения).
  • Маркирование Backspace unindents (Удаление структурного отступа при нажатии Backspace - состояние является стандартным) - курсор располагается на пустой строке или первом отличном от пробела символе строки. Нажатие клавиши Backspace приводит к выравниванию строки по предшествующему структурному уровню.
  • Если маркирована кнопка Cursor through tabs (Перемещать курсор по меткам табуляции), нажатие клавиши управления курсором будет перемещать курсор к середине табуляционных меток; в ином случае при перемещении курсора через метку табуляции курсор перемещается на несколько столбцов.
Tab Size 8 (Размер табуляционной метки 8)

Если в данном блоке диалога маркирована кнопка Use tab character (Использовать символы табуляции), и нажата клавишу Tab (Табуляция), система Borland C++ вставляет в файл символ табуляции, а курсор перемещается к очередной метке табуляции. Блок ввода Tab Size (Размер табуляционной метки) позволяет указывать число символов, на которое должен перемещаться курсор на каждой табуляционной метке. Допустимыми значениями являются значения в диапазоне от 2 до 16; по умолчанию используется значение 8. После изменения размера табуляционной метки, система Borland C++ произведет регенерацию файла, в результате чего он будет отображен с тем размером табуляционных меток, который был выбран. Это новое значение табуляционной метки можно сохранить в файле конфигурации, выбрав команду Options\Save Options (Параметры\Сохранить параметры).

Default Extension CPP (Стандартное расширение имени CPP)

Блок ввода Default Extension (Стандартное расширение имени) позволяет указать системе Borland C++, какое расширение имени файла следует использовать в качестве стандартного при компиляции и загрузке исходного кода.

Блок диалога Mouse (Параметры работы мыши)

Right Mouse Button (Правая кнопка мыши)
( ) Nothing (Ничего)
(∙) Topic search (Поиск темы)
( ) Go to cursor (Выполнить до позиции курсора)
( ) Breakpoint (Точка приостанова)
( ) Inspect (Инспекция)
( ) Evaluate (Вычислить)
( ) Add watch (Добавить выражение просмотра)

Селективная кнопка Right Mouse Button (Правая кнопка мыши) определяет то действие, которое будет выполняться при нажатии правой кнопки мыши (или, если маркирована триггерная кнопка инвертирования позиций кнопок мыши, левой кнопки). По умолчанию выполняется действие Topic search (Поиск темы). Перечень действий, выполняемых правой кнопкой мыши при маркировании любой кроме Nothing (Ничего) селективной кнопки (выбор селективной кнопки означает то же, что и выбор команды):

  • Topic search (Поиск темы) - Help\Topic Search (Подсказка\Поиск темы)
  • Go to cursor(Выполнить до позиции курсора) - Run\Go to cursor (Выполнить\Выполнить до позиции курсора)
  • Breakpoint(Точка приостанова) - Debug\Toggle Breakpoint (Отладка\Установить точку приостанова)
  • Inspect (Инспекция) - Debug\Inspect (Отладка\Инспекция)
  • Evaluate (Вычислить) - Debug\Evaluate (Отладка\Вычислить)
  • Add watch(Добавить выражение просмотра) - Debug\Watches\Add Watch (Отладка\Выражения просмотра\Добавить выражение просмотра)

В блоке Mouse Double Click (Двойное нажатие кнопки мыши) можно изменить позицию ползунка управляющей полосы, чтобы отрегулировать с помощью клавиш управления курсором скорость нажатия кнопок мыши, при которой данное нажатие будет рассматриваться как двойное. Перемещение ползунка в направлении слова Fast (Быстро) означает, что система Borland C++ требует для распознавания двойного нажатия кнопки мыши более короткого интервала времени между нажатиями. Перемещение ползунка в направлении слова Slow (Медленно) означает, что система Borland C++ по-прежнему будет распознавать двойное нажатие кнопки даже в том случае, если допускаете удлинение интервала времени между двумя нажатиями кнопки. Если хотите поэкспериментировать с различными значениями этого параметра, можете поупражняться в двойном нажатии кнопки Test (Проверка), которая расположена над управляющей полосой. Когда двойное нажатие этой кнопки прошло успешно, происходит подсвечивание этой кнопки цветовым (или световым) атрибутом.

[   ] Reverse mouse buttons (Инвертировать действия кнопок мыши)
Маркирование триггерной кнопки Reverse mouse buttons (Инвертировать действия кнопок мыши) приводит к тому, что активной кнопкой мыши становится правая кнопка вместо левой. В действительности переключения кнопок мыши не произойдет до тех пор, пока не выбрана кнопка OK (Выполнить).
