Для конспекта

 

Использование функций. Перегрузка функций. Рекурсивная функция.

  1. Перегруженные функции - несколько функций с одним и тем же именем, но с разными наборами параметров. 
    	// перегруженные функции Input - ввод числовых значений
	float Input(char *str)		// вещественные числа
	{	float arg;
		cout << str; cin >> arg;
		return arg;
	}
	int Input()			// целые числа
	{	int arg;
		cout << "int x = "; cin >> arg;
		return arg;
	}

  2. Рекурсивная функция - функция, которая вызывает саму себя или непосредственно, или косвенно через другую функцию. 
    	// рекурсивная функция fibonacci вызывает саму себя
	unsigned long fibonacci(unsigned long n)
	{
		if(n == 0 || n == 1) 
			return n;
		else 
			return fibonacci(n - 1) + fibonacci(n - 2);
	}

  3. Порождение все новых копий рекурсивной подпрограммы до выхода на граничное условие называется рекурсивным спуском.

  4. Максимальное количество копий рекурсивной подпрограммы, которое одновренно может находиться в памяти компьютера, называется глубиной рекурсии.

  5. Завершение работы рекурсивных подпрограмм, вплоть до самой первой, инициировавшей рекурсивные вызовы, называется рекурсивным подъёмом.

  6. Встраиваемая (inline) функция позволяет компилятору сгенерировать копию кода функции в соответствующем месте программы, чтобы избежать вызова этой функции.

  7. Имеются два способа обращения к функциям: вызов по значению, вызов по ссылке.

  8. В случае вызова по ссылке оператор вызова дает вызываемой функции возможность прямого доступа к передаваемым данным, а также возможность изменения этих данных.

  9. Ссылочный параметр - это псевдоним соответствующего аргумента.

  10. Аргумент по умолчанию - это значение, которое присваивается параметру функции при ее вызове, если при этом соответствующий аргумент функции не указан. 
    	// встраиваемая функция boxVol с аргументами по умолчании l=1, w=1, h=1
	inline int boxVol(int l=1, int w=1, int h=1)
	{
		return l*w*h;
	}
